# CK 仓库系统后续开发路线图（面向业务管理者）

本文档是给“不懂代码也能看懂”的执行计划。

## 一、项目目标

建立一套可持续、可追溯的人效分析体系：
- 精确记录每位人员在每个任务上的有效工时；
- 与 WMS 产出数据自动对接；
- 输出可用于管理决策的日报/周报/月报。

---

## 二、分阶段交付

## Phase 1：工时可信化（先做）

**目标**：保证工时数据“可算、可信、可追责”。

交付项：
1. 事件配对规则文档（join/leave）。
2. 每日工时明细（按人、任务、session）。
3. 异常清单（缺失 leave、重复 join、超长工时）。

验收标准：
- 任意抽样人员可回溯到完整作业轨迹。
- 工时汇总可与现场班次记录基本一致。

---

## Phase 2：WMS 数据打通（重点）

**目标**：把“投入工时”和“产出结果”放到同一张数据表。

交付项：
1. WMS 字段映射（字段名、业务含义、时间口径）。
2. 工时-产出关联规则（按日期 / 波次 / 任务）。
3. 合并后的分析明细（可导出 CSV）。

验收标准：
- 每天可自动产出“每人每任务产出效率”。
- 管理人员可解释每个指标来源。

---

## Phase 3：管理看板与预警（提升）

**目标**：用于日常管理决策和异常预警。

交付项：
1. 班组/个人对比报表。
2. 趋势图（天/周/月）。
3. 异常预警（低效、异常工时、产出偏差）。

验收标准：
- 每日可在 5 分钟内查看核心指标。
- 可定位到“低效发生在谁、哪个任务、哪个时段”。

---

## 三、建议的核心指标定义（首版）

1. 工时（小时） = 同一人同一任务 join->leave 的累计时长。
2. 人效（件/小时） = 任务件数 ÷ 任务工时。
3. 人效（单/小时） = 任务单量 ÷ 任务工时。
4. 出勤有效率 = 有效工时 ÷ 在岗总时长。
5. 异常率 = 异常记录数 ÷ 总记录数。

---

## 四、你需要提供的业务资料

为了尽快落地，请你准备：

1. 一份最近 3~7 天的 WMS 导出样例（可脱敏）。
2. 你当前任务分类与中文名称（PICK/PACK/RELABEL 等）。
3. 每个指标你希望的“目标值”（例如 PICK >= 120 件/小时）。
4. 班组组织结构（组长、成员）。
